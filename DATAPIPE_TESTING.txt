DATAPIPE / OSF TESTING
======================

DataPipe experiment ID embedded in experiment.js:
Ocl82q2wBw9q

Normal upload test:
1. Run build_test_matrices.bat if the Excel source changed.
2. Run run_local_server.bat.
3. Open one of these:
   http://localhost:8000/?matrix=TEST_01
   http://localhost:8000/?matrix=TEST_02
4. Finish the experiment.
5. During the final step, keep the page open while it says data are being uploaded.
6. A success page should appear.
7. Check the connected OSF Data component for a new CSV file.

Local-only test without uploading:
http://localhost:8000/?matrix=TEST_01&no_upload=1

Behavior:
- Upload success: success page; no participant action needed.
- Upload failure: the experiment attempts a local CSV download and shows a manual backup-download button.
- Every filename contains design ID, subject ID, session timestamp, and a random suffix, so names should be unique.

Important:
- Internet access is required for DataPipe upload.
- Data collection must remain enabled in the DataPipe dashboard.
- If the session limit is reached, DataPipe will reject new uploads until the limit is increased.
